Project Overview

Flatland Business Advisor is a sophisticated business tool that helps entrepreneurs in the fictional world of Flatland manage their ventures. This application demonstrates my advanced skills in DOM manipulation, interactive data visualization, and complex business logic implementation. Users can input business metrics, receive financial forecasts, and visualize their business health through interactive charts and reports.

Project Goals

  • Create a sophisticated business analytics tool with an intuitive interface
  • Implement advanced DOM manipulation for dynamic content rendering
  • Develop interactive data visualization components from scratch
  • Demonstrate complex business logic implementation in JavaScript
  • Build a modular, scalable application architecture
  • Create a responsive design appropriate for business professionals

Technologies Used

  • HTML5 for semantic structure
  • CSS3 with CSS Grid and Flexbox for layout
  • Vanilla JavaScript (ES6+) with no external libraries
  • Custom-built charts and visualization components
  • Local Storage for saving user data
  • Custom form validation and data processing
  • GitHub for version control
  • Github pages and Netlify for deployment

4. Custom Data Visualization Components

Rather than relying on external libraries, I built custom visualization components from scratch using JavaScript and SVG. This approach allowed for complete control over the appearance and behavior of the charts while demonstrating my ability to work with complex visual data representation. 

function createPieChart(containerId, data) {
  const container = document.getElementById(containerId);
  const svg = document.createElementNS('http://www.w3.org/2000/svg', 'svg');
  svg.setAttribute('viewBox', '0 0 100 100');
  svg.setAttribute('class', 'pie-chart');
  
  const radius = 50;
  const centerX = 50;
  const centerY = 50;
  
  // Calculate total for percentages
  const total = data.reduce((sum, item) => sum + item.value, 0);
  
  let startAngle = 0;
  
  // Create pie slices
  data.forEach((item, index) => {
    const percentage = item.value / total;
    const endAngle = startAngle + (percentage * 2 * Math.PI);
    
    // Calculate path coordinates
    const x1 = centerX + radius * Math.cos(startAngle);
    const y1 = centerY + radius * Math.sin(startAngle);
    const x2 = centerX + radius * Math.cos(endAngle);
    const y2 = centerY + radius * Math.sin(endAngle);
    
    // Create path element for slice
    const path = document.createElementNS('http://www.w3.org/2000/svg', 'path');
    
    // Path data
    const largeArcFlag = percentage > 0.5 ? 1 : 0;
    const pathData = `
      M ${centerX},${centerY}
      L ${x1},${y1}
      A ${radius},${radius} 0 ${largeArcFlag} 1 ${x2},${y2}
      Z
    `;
    
    path.setAttribute('d', pathData);
    path.setAttribute('fill', item.color);
    path.setAttribute('data-label', item.label);
    path.setAttribute('data-value', item.value);
    
    // Add event listeners for interactivity
    path.addEventListener('mouseover', showTooltip);
    path.addEventListener('mouseout', hideTooltip);
    
    svg.appendChild(path);
    
    // Update start angle for next slice
    startAngle = endAngle;
  });
  
  // Add legend and labels
  createChartLegend(container, data);
  
  container.appendChild(svg);
}

6. Form Validation and User Input Processing

I implemented comprehensive form validation to ensure data integrity and provide a smooth user experience. The validation includes real-time feedback, error handling, and data sanitization before processing. 

function validateBusinessForm(formData) {
  const errors = {};
  
  // Validate revenue (must be a positive number)
  if (!formData.revenue || isNaN(formData.revenue) || formData.revenue < 0) {
    errors.revenue = 'Revenue must be a positive number';
  }
  
  // Validate expenses (must be a positive number)
  if (!formData.expenses || isNaN(formData.expenses) || formData.expenses < 0) {
    errors.expenses = 'Expenses must be a positive number';
  }
  
  // Validate customer count (must be a positive integer)
  if (!formData.customers || isNaN(formData.customers) || 
      formData.customers < 0 || !Number.isInteger(Number(formData.customers))) {
    errors.customers = 'Customer count must be a positive whole number';
  }
  
  // Validate market size (must be >= customer count)
  if (!formData.marketSize || isNaN(formData.marketSize) || 
      formData.marketSize < formData.customers) {
    errors.marketSize = 'Market size must be at least equal to your customer count';
  }
  
  // Validate growth rate (must be between -100% and 1000%)
  if (!formData.growthRate || isNaN(formData.growthRate) || 
      formData.growthRate < -100 || formData.growthRate > 1000) {
    errors.growthRate = 'Growth rate must be between -100% and 1000%';
  }
  
  return {
    isValid: Object.keys(errors).length === 0,
    errors
  };
}

7. Local Storage Integration

To enhance the user experience, I implemented local storage functionality to save user data between sessions. This allows users to return to their previous work without needing to re-enter data. 

const StorageManager = {
  saveBusinessData(businessData) {
    try {
      // Add timestamp for version control
      const dataToSave = {
        ...businessData,
        lastUpdated: new Date().toISOString()
      };
      
      // Convert to JSON string and save
      localStorage.setItem('flatlandBusinessData', JSON.stringify(dataToSave));
      return true;
    } catch (error) {
      console.error('Error saving business data:', error);
      return false;
    }
  },
  
  loadBusinessData() {
    try {
      const savedData = localStorage.getItem('flatlandBusinessData');
      if (!savedData) return null;
      
      // Parse JSON and return
      return JSON.parse(savedData);
    } catch (error) {
      console.error('Error loading business data:', error);
      return null;
    }
  },
  
  clearBusinessData() {
    try {
      localStorage.removeItem('flatlandBusinessData');
      return true;
    } catch (error) {
      console.error('Error clearing business data:', error);
      return false;
    }
  }
};

Challenges & Solutions

Challenge: Building Custom Chart Components

Creating responsive, interactive data visualization components from scratch without relying on external libraries presented significant challenges in terms of mathematics, SVG manipulation, and browser compatibility.

Solution:

I approached this by first thoroughly researching SVG fundamentals and data visualization best practices. I then broke down each chart type into smaller components, developing and testing them incrementally. For complex mathematical calculations, I created utility functions that could be reused across different chart types. Finally, I implemented a responsive design system that adjusts chart dimensions and text elements based on container size.

Challenge: Implementing Complex Business Logic

Developing accurate business forecasting algorithms and financial metrics calculations required detailed understanding of business principles while keeping the code maintainable and efficient.

Solution:

I created a modular approach to business logic, breaking down complex calculations into smaller, testable functions. Each business metric was implemented as a separate module with clear input and output specifications. This approach allowed for easier testing and validation of results. I also incorporated data validation at multiple levels to ensure accurate calculations even with edge-case inputs.

Challenge: Ensuring Cross-Browser Compatibility

The application's heavy reliance on advanced JavaScript features and SVG manipulation led to compatibility issues across different browsers, particularly older ones.

Solution:

I implemented feature detection rather than browser detection to ensure the application would work correctly across different browsers. For critical features, I created fallback functionality that would provide a simpler but still functional experience in browsers with limited support. I also used a combination of polyfills and transpilation to ensure modern JavaScript features would work consistently across browsers.
