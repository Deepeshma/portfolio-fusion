Rock Paper Scissors Game

Live Demo Source Code

Project Overview

This interactive Rock Paper Scissors game demonstrates my ability to create engaging web applications using JavaScript. The project showcases DOM manipulation, event handling, animations, and game logic implementation. Players can compete against a computer opponent with real-time score tracking and visual feedback for each round's outcome.

Project Goals

  • Create an interactive game with intuitive user interface
  • Implement core JavaScript concepts including DOM manipulation
  • Add engaging animations and visual feedback
  • Build a responsive design that works on all devices
  • Practice object-oriented programming principles

Technologies Used

  • HTML5 for structure
  • CSS3 with animations and transitions
  • Vanilla JavaScript (ES6+)
  • Local Storage API for saving game statistics
  • CSS Grid and Flexbox for responsive layout
  • GitHub for version control
  • Github pages and Netlify for deployment

4. JavaScript Game Logic

The core game logic was implemented using modern JavaScript (ES6+). I structured the code using object-oriented principles for better organization and maintainability. The game includes several key features:

  • Random computer selection algorithm
  • Win determination logic
  • Score tracking and storage
  • Visual feedback for game outcomes
  • Animation controls for interactive elements
function playRound(userChoice) {
  const computerChoice = getComputerChoice();
  const result = determineWinner(userChoice, computerChoice);
  
  displayChoices(userChoice, computerChoice);
  updateScore(result);
  animateResult(result, userChoice, computerChoice);
  
  // Store game statistics
  saveGameToHistory(userChoice, computerChoice, result);
}

function determineWinner(userChoice, computerChoice) {
  if (userChoice === computerChoice) {
    return "tie";
  }
  
  const winConditions = {
    rock: "scissors",
    paper: "rock",
    scissors: "paper"
  };
  
  return winConditions[userChoice] === computerChoice ? "win" : "lose";
}

5. Local Storage Implementation

To enhance the game experience, I implemented local storage to save the player's game history and statistics. This allows players to track their performance over time and adds a layer of engagement. 

function saveGameToHistory(userChoice, computerChoice, result) {
  let gameHistory = JSON.parse(localStorage.getItem('rpsHistory')) || [];
  
  const gameRecord = {
    timestamp: Date.now(),
    userChoice,
    computerChoice,
    result
  };
  
  gameHistory.push(gameRecord);
  
  // Keep only the most recent 50 games
  if (gameHistory.length > 50) {
    gameHistory = gameHistory.slice(-50);
  }
  
  localStorage.setItem('rpsHistory', JSON.stringify(gameHistory));
  updateStatistics();
}

Challenges & Solutions

Challenge: Creating Engaging Animations

Developing animations that would enhance gameplay without being distracting or causing performance issues on mobile devices was challenging.

Solution:

I used CSS transitions and keyframe animations optimized for performance. By limiting animations to transform and opacity properties, I was able to create smooth effects that run well even on lower-end devices. I also implemented conditional animation complexity based on device capabilities.

Challenge: Creating a Fair Computer Opponent

I wanted to create a computer opponent that felt fair and challenging without being predictable.

Solution:

Rather than using pure randomness, I implemented a weighted random selection algorithm that adjusts based on the player's previous choices and success rate. This creates a more engaging opponent that adapts to the player's pattern while still maintaining fairness.
