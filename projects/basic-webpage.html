Urban Brew Cafe

Live Demo Source Code

Project Overview

Urban Brew Cafe is a foundational HTML & CSS project designed to showcase my ability to create a clean, responsive website with a focus on semantic markup and accessibility. This fictional cafe website demonstrates my understanding of core web development principles and attention to detail in implementing a modern design.

Project Goals

  • Create a visually appealing website with proper semantic HTML structure
  • Implement responsive design principles for all device sizes
  • Ensure WCAG accessibility standards are met
  • Optimize performance for fast loading speeds
  • Build a user-friendly navigation experience

Technologies Used

  • HTML5 semantic elements
  • CSS3 with Flexbox and Grid
  • CSS custom properties (variables)
  • CSS animations and transitions
  • Responsive design with media queries
  • GitHub for version control
  • Github pages for deployment

Development Process

1. Planning and Design

I began by creating wireframes and design mockups for the cafe website, focusing on creating a warm, inviting atmosphere that would appeal to coffee lovers. The color palette was chosen to reflect the rich tones of coffee and create a cozy ambiance.

2. HTML Structure

The website was built using semantic HTML5 elements to ensure proper structure and accessibility. Section elements were used for major page divisions, with appropriate ARIA attributes added where necessary to enhance screen reader compatibility. 

<header>
                                    <nav>
                                        <ul class="nav-links">
                                        <li><a href="#home" class="active">Home</a></li>
                                        <li><a href="#menu">Menu</a></li>
                                        <li><a href="#about">About Us</a></li>
                                        <li><a href="#contact">Contact</a></li>
                                        </ul>
                                    </nav>
                                    </header>

3. CSS Implementation

I implemented a mobile-first approach with CSS, ensuring that the site looks great on all device sizes. CSS custom properties were used to maintain a consistent color scheme and simplify future updates. Flexbox and Grid layouts were employed to create responsive sections that adapt beautifully to different screen sizes. 

:root {
                                --primary-color: #5e3a1a;
                                --secondary-color: #d4a76a;
                                --background-color: #f9f5f0;
                                --text-color: #333333;
                                --accent-color: #b85c38;
                                }

                                .menu-grid {
                                display: grid;
                                grid-template-columns: repeat(auto-fill, minmax(300px, 1fr));
                                gap: 2rem;
                                }

4. Responsive Design

Media queries were strategically implemented to ensure optimal viewing experience across devices. The navigation menu transforms into a hamburger menu on mobile devices, and content sections adjust their layout for different screen sizes.

5. Testing and Optimization

I tested the website across multiple browsers and devices to ensure consistent performance. Accessibility was verified using tools like WAVE and Lighthouse, and performance optimizations were made to improve loading speeds, including image optimization and CSS minification.

Challenges & Solutions

Challenge: Creating an Engaging Menu Display

Designing a menu section that was both visually appealing and functional on all device sizes required careful consideration of layout and typography.

Solution:

I implemented a responsive grid layout with CSS Grid that automatically adjusts the number of columns based on screen width. Each menu item was designed as a card with subtle hover effects to enhance interactivity.

Challenge: Navigation Experience on Mobile

Ensuring a smooth navigation experience on mobile devices without cluttering the interface was a key consideration.

Solution:

I created a hamburger menu that expands into a full-screen navigation overlay on mobile devices, with smooth animations to enhance the user experience. The menu items were given ample tap targets for improved usability.

Outcomes & Lessons Learned

This project reinforced my understanding of semantic HTML structure and responsive design principles. I gained valuable experience implementing accessibility features and optimizing website performance. Key takeaways include:

  • The importance of planning the HTML structure before diving into CSS styling
  • How to effectively use CSS custom properties to maintain consistent design systems
  • Techniques for creating responsive layouts that work across all device sizes
  • Best practices for implementing accessible navigation patterns

The project received positive feedback for its clean design and smooth user experience, particularly the responsive behavior and attention to detail in typography and spacing.
