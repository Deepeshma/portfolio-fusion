RSS Reader

Project Overview

The RSS Reader is a modern web application that aggregates and displays content from multiple RSS feeds in a clean, user-friendly interface. This project demonstrates my proficiency with asynchronous JavaScript, API integration, and dynamic content rendering. Users can subscribe to multiple feeds, filter content by source or category, search for specific articles, and save favorites for later reading.

Project Goals

  • Create a modern RSS aggregator with an intuitive user interface
  • Implement asynchronous data fetching using the Fetch API
  • Build advanced filtering and search functionality
  • Develop a responsive design for all device types
  • Practice working with external APIs and XML parsing
  • Implement persistent data storage for user preferences

Technologies Used

  • HTML5 for semantic structure
  • CSS3 with CSS Grid and Flexbox for responsive layout
  • JavaScript (ES6+) for application logic
  • Fetch API for asynchronous data retrieval
  • DOMParser for XML/RSS parsing
  • Local Storage API for saving user preferences
  • CORS proxy for handling cross-origin requests
  • GitHub for version control
  • Github Pages for deployment

3. Implementing Fetch API for RSS Retrieval

One of the key challenges was fetching RSS feeds, which often encounter CORS restrictions. I implemented a solution using a CORS proxy service to handle cross-origin requests while maintaining security. 

async function fetchFeed(feedUrl) {
  try {
    // Use CORS proxy to avoid cross-origin issues
    const proxyUrl = 'https://api.allorigins.win/raw?url=';
    const response = await fetch(`${proxyUrl}${encodeURIComponent(feedUrl)}`);
    
    if (!response.ok) {
      throw new Error(`Failed to fetch feed: ${response.status} ${response.statusText}`);
    }
    
    const data = await response.text();
    return data;
  } catch (error) {
    console.error('Error fetching feed:', error);
    throw error;
  }
}

async function processFeed(feed) {
  try {
    const xmlData = await fetchFeed(feed.url);
    const parser = new DOMParser();
    const xmlDoc = parser.parseFromString(xmlData, 'text/xml');
    
    // Determine feed type (RSS vs Atom) and parse accordingly
    const feedType = determineFeedType(xmlDoc);
    let articles;
    
    if (feedType === 'atom') {
      articles = parseAtomFeed(xmlDoc, feed);
    } else {
      articles = parseRSSFeed(xmlDoc, feed);
    }
    
    return articles;
  } catch (error) {
    console.error(`Error processing feed ${feed.url}:`, error);
    return [];
  }
}

4. XML Parsing and Normalization

RSS feeds come in various formats (RSS 2.0, Atom, etc.), so I created parsers that could handle different feed types and normalize the data into a consistent format for the application. 

function parseRSSFeed(xmlDoc, feedInfo) {
  const items = xmlDoc.querySelectorAll('item');
  const articles = [];
  
  items.forEach(item => {
    // Extract basic item information
    const title = getElementText(item, 'title');
    const link = getElementText(item, 'link');
    const description = getElementText(item, 'description');
    const pubDateText = getElementText(item, 'pubDate');
    
    // Parse date (handling various date formats)
    const pubDate = parseFeedDate(pubDateText);
    
    // Extract categories/tags
    const categories = [];
    item.querySelectorAll('category').forEach(category => {
      categories.push(category.textContent.trim());
    });
    
    // Extract media if available
    let mediaUrl = '';
    const mediaContent = item.querySelector('media\\:content, content');
    if (mediaContent && mediaContent.getAttribute('url')) {
      mediaUrl = mediaContent.getAttribute('url');
    } else {
      // Try to extract image from enclosure
      const enclosure = item.querySelector('enclosure');
      if (enclosure && enclosure.getAttribute('type')?.startsWith('image/')) {
        mediaUrl = enclosure.getAttribute('url');
      } else {
        // Try to extract first image from description
        const imgMatch = description.match(/]+src="([^">]+)"/);
        if (imgMatch) {
          mediaUrl = imgMatch[1];
        }
      }
    }
    
    articles.push({
      id: generateUniqueId(link),
      title,
      link,
      description: cleanDescription(description),
      pubDate,
      source: {
        id: feedInfo.id,
        title: feedInfo.title,
        url: feedInfo.url
      },
      categories,
      mediaUrl,
      isRead: false,
      isFavorite: false
    });
  });
  
  return articles;
}

6. Implementing Search and Filter Functionality

To enhance the user experience, I implemented robust search and filtering capabilities that allow users to quickly find relevant content within their feeds. 

function searchArticles(articles, query) {
  if (!query || query.trim() === '') {
    return articles;
  }
  
  const searchTerms = query.toLowerCase().trim().split(' ').filter(term => term.length > 0);
  
  return articles.filter(article => {
    // Search in title, description, and categories
    const titleMatch = searchTerms.some(term => 
      article.title.toLowerCase().includes(term)
    );
    
    const descriptionMatch = searchTerms.some(term => 
      article.description.toLowerCase().includes(term)
    );
    
    const categoryMatch = article.categories.some(category => 
      searchTerms.some(term => category.toLowerCase().includes(term))
    );
    
    return titleMatch || descriptionMatch || categoryMatch;
  });
}

function applyFilters(articles, filters) {
  let filteredArticles = [...articles];
  
  // Filter by source
  if (filters.sources && filters.sources.length > 0) {
    filteredArticles = filteredArticles.filter(article => 
      filters.sources.includes(article.source.id)
    );
  }
  
  // Filter by category
  if (filters.categories && filters.categories.length > 0) {
    filteredArticles = filteredArticles.filter(article => 
      article.categories.some(category => 
        filters.categories.some(filter => category.toLowerCase().includes(filter.toLowerCase()))
      )
    );
  }
  
  // Filter by read/unread status
  if (filters.readStatus === 'read') {
    filteredArticles = filteredArticles.filter(article => article.isRead);
  } else if (filters.readStatus === 'unread') {
    filteredArticles = filteredArticles.filter(article => !article.isRead);
  }
  
  // Filter by favorites
  if (filters.onlyFavorites) {
    filteredArticles = filteredArticles.filter(article => article.isFavorite);
  }
  
  // Sort articles
  filteredArticles.sort((a, b) => {
    if (filters.sortBy === 'date') {
      return b.pubDate - a.pubDate; // Newest first
    } else if (filters.sortBy === 'source') {
      return a.source.title.localeCompare(b.source.title);
    }
    return 0;
  });
  
  return filteredArticles;
}

7. Implementing Local Storage for Persistence

To enhance the user experience, I implemented local storage functionality to save user preferences, feed subscriptions, read status, and favorites between sessions. 

const StorageManager = {
  saveFeeds(feeds) {
    try {
      localStorage.setItem('rssReader_feeds', JSON.stringify(feeds));
      return true;
    } catch (error) {
      console.error('Error saving feeds to local storage:', error);
      return false;
    }
  },
  
  loadFeeds() {
    try {
      const savedFeeds = localStorage.getItem('rssReader_feeds');
      return savedFeeds ? JSON.parse(savedFeeds) : [];
    } catch (error) {
      console.error('Error loading feeds from local storage:', error);
      return [];
    }
  },
  
  saveArticleStatus(articleId, status) {
    try {
      // Get existing status objects
      const articleStatus = JSON.parse(localStorage.getItem('rssReader_articleStatus') || '{}');
      
      // Update status for this article
      articleStatus[articleId] = {
        ...articleStatus[articleId],
        ...status,
        lastUpdated: new Date().toISOString()
      };
      
      // Save back to storage
      localStorage.setItem('rssReader_articleStatus', JSON.stringify(articleStatus));
      return true;
    } catch (error) {
      console.error('Error saving article status:', error);
      return false;
    }
  },
  
  loadArticlesStatus() {
    try {
      const status = localStorage.getItem('rssReader_articleStatus');
      return status ? JSON.parse(status) : {};
    } catch (error) {
      console.error('Error loading article status:', error);
      return {};
    }
  },
  
  saveUserPreferences(preferences) {
    try {
      localStorage.setItem('rssReader_preferences', JSON.stringify(preferences));
      return true;
    } catch (error) {
      console.error('Error saving user preferences:', error);
      return false;
    }
  }
};

Challenges & Solutions

Challenge: Cross-Origin Resource Sharing (CORS)

One of the biggest challenges was dealing with CORS restrictions when fetching RSS feeds from different domains, which is a common security limitation in web browsers.

Solution:

I implemented a proxy-based solution that routes feed requests through a CORS-enabled API proxy. This approach maintains security while allowing the application to access feeds from any domain. I also implemented a fallback system that tries multiple proxy services in case one fails, ensuring reliable feed fetching.

Challenge: Handling Different Feed Formats

RSS feeds come in multiple formats (RSS 1.0, RSS 2.0, Atom) with different structures, making consistent parsing and normalization challenging.

Solution:

I created a feed format detection system that identifies the feed type before parsing. Each format has its own specialized parser that extracts the relevant data and normalizes it into a consistent internal format. This abstraction layer shields the rest of the application from needing to handle format-specific details, making the code more maintainable.

Challenge: Performance with Large Numbers of Articles

When users subscribe to multiple active feeds, the number of articles can quickly grow into the hundreds or thousands, causing performance issues during rendering and searching.

Solution:

I implemented a virtual scrolling technique that only renders articles currently visible in the viewport. This dramatically improves performance by reducing DOM elements and rendering work. Additionally, I implemented an indexing system for search operations that makes full-text search efficient even with thousands of articles.
